# 프레젠테이션 페이지 기능 정의서 (확정)

## 기능 개요

홈 페이지의 `Design to Commit` 발표 카드를 클릭하면 `/harness` 경로에서 슬라이드 형태의 발표 자료를 볼 수 있는 페이지를 제공한다.

슬라이드 레이아웃과 네비게이션 구조를 구현하고, 각 슬라이드 콘텐츠는 제목과 핵심 키워드 수준의 placeholder로 채운다. 발표 내용이 확정되면 상수 배열만 수정하여 교체할 수 있도록 설계한다.

## 기능 요구사항

- `/harness` 페이지는 발표용 슬라이드 화면으로 구성한다.
- 슬라이드는 16:9 비율의 고정 비율 박스로 화면 중앙에 배치한다.
- 사용자는 이전/다음 슬라이드로 이동할 수 있다.
- 키보드 방향키 `←`, `→`로 슬라이드를 이동할 수 있다.
- 슬라이드 박스 좌우 가장자리에 오버레이 방식으로 이전/다음 화살표 버튼을 제공한다.
- 슬라이드 하단 중앙에 `현재 / 전체` 형식의 인디케이터를 표시한다.
- 슬라이드 전환 시 좌우 슬라이드 애니메이션을 사용한다.
- `prefers-reduced-motion` 환경에서는 애니메이션 없이 즉시 교체한다.
- 첫 번째 슬라이드에서 이전 이동을 시도하면 아무 동작도 하지 않는다.
- 마지막 슬라이드에서 다음 이동을 시도하면 아무 동작도 하지 않는다.
- 슬라이드 데이터는 상수 배열로 관리한다.

## 초기 발표 정보

| 항목        | 내용                                                           |
| ----------- | -------------------------------------------------------------- |

| 경로        | `/harness`                                                     |

## 슬라이드 구성

각 슬라이드는 제목과 핵심 키워드 수준의 콘텐츠를 포함한다. 발표 내용 확정 후 상수 배열을 교체한다.

| 순서 | 슬라이드 역할       |
| ---- | ------------------- |
| 1    | 문제 제기           |
| 2    | 제안하는 파이프라인 |
| 3    | 각 기술 소개        |
| 4    | 현재 구현된 것      |
| 5    | 도전해보고 싶은 것  |
| 6    | 기대 효과           |
| 7    | 한 줄 요약          |

## UI / UX 요구사항

### 슬라이드 레이아웃

```
┌─────────────────────────────────────────────────────┐
│  ◀                  [슬라이드 콘텐츠]             ▶  │  ← 오버레이 화살표
│                                                     │
│                                                     │
│                      제목                           │
│                  · 키워드 1                         │
│                  · 키워드 2                         │
│                  · 키워드 3                         │
│                                                     │
└──────────────────────────────────────────────────────┘
                        3 / 7                           ← 하단 인디케이터
```

- 슬라이드 박스: `aspect-video`(16:9) 고정 비율, 화면 중앙 배치
- 화살표 버튼: 슬라이드 박스 좌우 가장자리 오버레이, 호버 시 강조
- 인디케이터: 슬라이드 박스 하단 중앙, `현재 / 전체` 형식

### 슬라이드 전환

- 방향: 다음 이동 시 좌로 슬라이드, 이전 이동 시 우로 슬라이드
- `prefers-reduced-motion` 환경에서는 즉시 교체

### 디자인

- 다크 모드 기반 화면 (홈 페이지 다크 기본값 그대로 유지)
- 디자인 시스템 토큰 사용
- 발표용 타이포그래피: 충분히 큰 글자 크기와 여백

### 네비게이션

- `ArrowRight` → 다음 슬라이드
- `ArrowLeft` → 이전 슬라이드
- 첫/마지막 슬라이드에서 경계 동작 없음 (조용히 무시)
- 마우스 클릭으로도 이동 가능

## 비고

- 초기 버전에서는 서버 연동을 제외한다.
- PDF Export, Speaker Note, 발표자 모드는 구현하지 않는다.
- 모바일 최적화는 우선순위에서 제외한다.
- 발표 검색, 필터링, 공유 기능은 구현하지 않는다.
