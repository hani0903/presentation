# 홈 페이지 기능 정의서 (확정)

## 기능 개요

개인 발표 자료 보관소(Presentation Archive). 준비된 발표 자료로 이동할 수 있는 홈 페이지를 제공한다.

초기 버전에서는 발표 자료가 1개만 존재하지만, 향후 여러 발표 자료를 추가할 수 있도록 카드 목록 기반 구조로 설계한다.

## 기능 요구사항

- 사이트 이름(`Presentation Archive`)과 간단한 소개 문구를 보여준다.
- 발표 카드는 제목, 한 줄 설명, 발표 주제 키워드(배지 형태)를 포함한다.
- 발표 카드 클릭 시 `/harness` 경로를 **새 탭**으로 연다.
- 발표 카드 목록은 상수 배열로 관리하여 향후 발표 추가가 쉽도록 한다.
- 다크모드 토글을 우상단에 배치하며, **기본값은 다크 모드**로 고정한다.
- `/harness` 라우트에 placeholder 페이지를 제공한다.

## 초기 발표 카드

| 항목   | 내용                                                           |
| ------ | -------------------------------------------------------------- |

| 설명   | Figma에서 커밋까지 이어지는 AI 기반 컴포넌트 자동화 파이프라인 |
| 경로   | `/harness`                                                     |

## UI / UX 요구사항

### 레이아웃

- 전체 화면 수직 중앙 정렬 구조 (카드 1개 상태에서 자연스럽도록)
- 사이트 이름과 소개 문구를 상단 히어로 영역으로, 그 아래 카드 배치
- 향후 카드가 여러 개로 늘어나도 그리드 구조로 자연스럽게 확장 가능하도록 설계

```
┌─────────────────────────────[🌙]─┐
│                                  │
│       Presentation Archive       │
│          한 줄 소개 문구          │
│                                  │
│              [카드]              │
│                                  │
└──────────────────────────────────┘
```

### 다크모드 토글

- 우상단 고정 배치
- 기본값: 다크 모드 (localStorage에 저장값 없을 경우 dark로 초기화)
- 라이트/다크 모드 전환 가능

### 발표 카드

- 키워드는 배지(Badge/Chip) 형태로 표시
- 클릭 시 새 탭(`target="_blank"`)으로 이동
- 호버 시 시각적 피드백(배경색 변화 등)

### 디자인

- 디자인 시스템 토큰 사용
- 라이트/다크 모드 대응
- 데스크톱 발표 환경 우선

## 비고

- 초기 버전에서는 로그인, 회원가입, 서버 연동을 제외한다.
- 홈 페이지는 정적 UI 중심으로 구현한다.
- 발표 목록 데이터는 상수 배열로 관리한다.
- 발표 검색, 필터링, 카테고리 기능은 구현하지 않는다.
