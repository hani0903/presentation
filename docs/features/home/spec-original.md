# 홈 페이지 기능 정의서

## 기능 개요

사용자가 사이트에 처음 진입했을 때 사이트의 목적을 이해하고, 준비된 발표 자료로 이동할 수 있는 홈 페이지를 제공한다.

초기 버전에서는 발표 자료가 1개만 존재하지만, 향후 여러 발표 자료를 추가할 수 있도록 카드 목록 기반 구조로 설계한다.

## 기능 요구사항

- 사이트 이름과 간단한 소개 문구를 보여준다.
- 발표 자료를 확인할 수 있는 진입점을 제공한다.
- 현재는 `Design to Commit` 발표 카드 1개를 노출한다.
- 발표 카드는 제목, 한 줄 설명, 발표 주제 키워드를 포함한다.
- 발표 카드 클릭 시 `/harness` 경로로 이동한다.
- 발표 카드 목록은 상수 배열로 관리하여 향후 발표 추가가 쉽도록 한다.

## 초기 발표 카드

| 항목   | 내용                                                           |
| ------ | -------------------------------------------------------------- |
| 제목   | Design to Commit                                               |
| 설명   | Figma에서 커밋까지 이어지는 AI 기반 컴포넌트 자동화 파이프라인 |
| 경로   | `/harness`                                                     |
| 키워드 | Figma, Design System, Claude Harness, Storybook, Husky         |

## UI / UX 요구사항

- 첫 화면에서 발표 사이트임을 즉시 이해할 수 있어야 한다.
- 데스크톱 발표 환경을 우선 고려한다.
- 카드 1개만 있어도 어색하지 않도록 중앙 정렬 또는 히어로 섹션 형태로 구성한다.
- 향후 카드가 여러 개로 늘어나도 자연스럽게 확장 가능한 그리드 구조를 고려한다.
- 디자인 시스템 토큰을 사용한다.
- 라이트/다크 모드에 대응한다.

## 비고

- 초기 버전에서는 로그인, 회원가입, 서버 연동을 제외한다.
- 홈 페이지는 정적 UI 중심으로 구현한다.
- 발표 목록 데이터는 상수 배열로 관리한다.
- 이번 버전에서는 발표 검색, 필터링, 카테고리 기능은 구현하지 않는다.
