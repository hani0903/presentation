# 프레젠테이션 페이지 기능 정의서

## 기능 개요

사용자가 발표 자료를 슬라이드 형식으로 볼 수 있는 프레젠테이션 페이지를 제공한다.

초기 버전에서는 `/harness` 경로에서 **Claude Harness를 활용한 AI 개발 워크플로우 구축** 발표를 보여준다.

이번 구현의 목적은 발표 콘텐츠 자체보다 **슬라이드 구조와 레이아웃 시스템을 구축하는 것**이다.

발표 자료는 공통 슬라이드 템플릿을 조합하여 구성하며, 이후 다양한 발표 콘텐츠를 동일한 구조 위에서 재사용할 수 있어야 한다.

---

## 기능 요구사항

- 발표는 여러 장의 슬라이드로 구성된다.
- 모든 슬라이드는 16:9 비율을 기준으로 렌더링한다.
- 슬라이드는 화면 중앙에 배치한다.
- 사용자는 이전/다음 슬라이드로 이동할 수 있다.
- 키보드 방향키(`←`, `→`)를 지원한다.
- 마우스 클릭 기반 네비게이션을 지원한다.
- 현재 슬라이드 번호와 전체 슬라이드 수를 표시한다.
- 슬라이드 전환은 과하지 않은 기본 애니메이션을 사용한다.
- `prefers-reduced-motion` 사용자를 고려한다.
- 다크 모드를 기본으로 사용한다.
- 발표 콘텐츠와 레이아웃 구조를 분리한다.
- 향후 다른 발표 자료도 동일한 슬라이드 구조를 재사용할 수 있어야 한다.

## 슬라이드 템플릿 구조

모든 슬라이드는 아래 템플릿 중 하나를 사용한다.

---

### 1. CoverSlide

발표의 시작 페이지에 사용한다.

#### 목적

발표 제목과 발표 정보를 전달한다.

#### 구조

```text
발표 제목

부제

발표자 정보
```

#### 레이아웃

- 중앙 정렬
- 가장 큰 타이포그래피 사용
- 발표의 첫인상을 담당

---

### 2. SectionSlide

대주제가 변경될 때 사용하는 구분 페이지다.

#### 목적

발표 흐름을 명확하게 구분한다.

#### 구조

```text
03

내가 구축하고 있는
Claude Harness

CLAUDE.md
Design System
Feature Planning
```

#### 레이아웃

- 중앙 정렬
- 콘텐츠 설명보다 챕터 전환에 집중
- 현재 섹션에서 다룰 내용을 요약 가능

---

### 3. ContentSlide

가장 기본적인 본문 슬라이드다.

#### 목적

개념 설명, 코드 소개, 사례 설명 등에 사용한다.

#### 구조

```text
03. Claude Harness

CLAUDE.md

본문 콘텐츠
```

#### 레이아웃

- 상단 제목 영역
- 하단 콘텐츠 영역
- 제목과 본문을 명확히 분리

#### 규칙

- 대주제는 작게 표시한다.
- 현재 슬라이드의 핵심 제목을 크게 표시한다.
- 한 슬라이드에는 하나의 핵심 메시지만 담는다.

---

### 4. ProcessSlide

워크플로우나 파이프라인을 설명할 때 사용한다.

#### 목적

단계적 흐름을 시각적으로 표현한다.

#### 예시

```text
spec-original
 ↓
Spec Interview
 ↓
spec-fixed
 ↓
PRD
 ↓
ADR
 ↓
Issue
 ↓
구현
 ↓
검증
```

#### 레이아웃

- 흐름 영역
- 설명 영역

2개 영역으로 구성할 수 있다.

---

### 5. SplitSlide

두 가지 내용을 비교하거나 병렬로 보여줄 때 사용한다.

#### 목적

현재 상태와 미래 계획, 장점과 단점 등을 비교한다.

#### 예시

```text
현재 검증 완료      |      다음 실험 계획
```

#### 레이아웃

- 좌우 2열 구조
- 두 영역은 동일한 중요도를 가진다.

## 콘텐츠 관리 방식

- 슬라이드 데이터는 상수 배열로 관리한다.
- 각 슬라이드는 고유 id를 가진다.
- 슬라이드는 템플릿 타입을 가진다.
- 슬라이드 내용은 데이터로 주입한다.
- 발표 콘텐츠와 레이아웃 구현은 분리한다.
- 향후 HTML, MDX, CMS 등의 콘텐츠 소스로 교체 가능해야 한다.

---

## UI / UX 요구사항

- 발표자가 쉽게 읽을 수 있어야 한다.
- 슬라이드는 padding 영역을 제외하고 최대한 넓게 사용한다.
- 제목은 충분히 큰 크기를 사용한다.
- 콘텐츠는 과도하게 길어지지 않도록 제한한다.
- 한 슬라이드는 하나의 핵심 메시지를 전달해야 한다.
- 발표 흐름을 따라가기 쉽도록 레이아웃 일관성을 유지한다.
- 디자인 시스템의 Typography, Color, Spacing 규칙을 따른다.

---

## 네비게이션 요구사항

- `ArrowRight` 입력 시 다음 슬라이드로 이동한다.
- `ArrowLeft` 입력 시 이전 슬라이드로 이동한다.
- 첫 번째 슬라이드에서 이전 이동 시 아무 동작도 하지 않는다.
- 마지막 슬라이드에서 다음 이동 시 아무 동작도 하지 않는다.
- 마우스 클릭으로도 이동할 수 있어야 한다.
- `현재 / 전체` 형식의 인디케이터를 제공한다.

---

## Out of Scope

이번 버전에서는 다음 기능을 구현하지 않는다.

- 서버 연동
- 발표 검색
- 발표 목록 필터링
- 발표 공유 기능
- Speaker Note
- 발표자 모드
- PDF Export
- 발표 녹화 기능
- 실시간 협업 기능
- 모바일 최적화
- 콘텐츠 편집 기능

---

## 비고

- 이번 버전의 핵심은 슬라이드 콘텐츠가 아니라 발표 레이아웃 시스템 구축이다.
- 발표용 UI 컴포넌트(CodeBlock, FlowDiagram, BulletList 등)는 별도 문서에서 정의한다.
- 이후 Presentation Component System을 추가하여 슬라이드 내부 콘텐츠를 컴포넌트 조합 방식으로 구성할 예정이다.
- Storybook 기반 문서화는 향후 실험 항목으로 다룬다.
